    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-storm-control" class="anchor" href="#storm-control" aria-hidden="true"><span class="octicon octicon-link"></span></a>storm-control</h1>

<p>This is a repository of code developed in the <a href="http://zhuang.harvard.edu">Zhuang Lab</a> for the acquisition of STORM movies.</p>

<h1>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h1>

<p>You will need Python and PyQt as well as a number of other libraries. Please see the Install.txt file in the hal4000 folder.</p>

<h1>
<a id="user-content-directory-layout" class="anchor" href="#directory-layout" aria-hidden="true"><span class="octicon octicon-link"></span></a>Directory Layout</h1>

<p>dave - This folder contains software (dave.py) that is used to control hal for automated acquisition of multiple images and/or STORM movies. Dave controls Hal via TCP/IP.</p>

<p>hal4000 - This folder contains the hal-4000 microscope control and image acquisition software (hal-4000.py).</p>

<p>sc_library - This folder contains the modules that are used in multiple different programs.</p>

<p>steve - This folder contains software (steve.py) that is used to take and assemble image mosaics. This is useful for array tomography experiments, among other things. Steve also controls Hal via TCP/IP.</p>

<p>zee-calibrator - This folder contains software (main.py) that is used to generate calibration curves for astigmatism based 3D STORM imaging. Calibration data acquired using Hal needs to be analyzed by STORM image analysis software like Insight3 (available by request from the Zhuang Lab) or 3D-DAOSTORM before it can be processed using zee-calibrator.</p>

<h1>
<a id="user-content-general-notes" class="anchor" href="#general-notes" aria-hidden="true"><span class="octicon octicon-link"></span></a>General notes</h1>

<ol class="task-list">
<li><p>This software is written primarily in Python with a few C helper libraries.</p></li>
<li><p>Doxygen documentation is available (for Dave, HAL and Steve), but you'll have to run doxygen to create it.</p></li>
<li><p>The different branches correspond to different setups. Ideally these are all modifications off the tip of master, but depending on update frequency some may lag a bit.</p></li>
<li><p>The software is provided "as is" in the hope that others might find it useful. While it is fairly stable and has been developed and used since 2009 in the Zhuang lab, we provide no gaurantee that any future changes that are made will maintain backwards compatibility with older versions.</p></li>
<li><p>We can only provide fairly limited support. You will probably have the most success adapting this software for your purposes if you are reasonably familiar with the Python programming language.</p></li>
<li><p>Questions should be addressed to Hazen Babcock (hbabcock <em>at</em> fas.harvard.edu).</p></li>
</ol>
